About

  • Connect to and view your local MySQL databases.
  • Query your database using natural language instead of writing tedious SQL queries. No need to ask in generic langugage, ask contextual questions specific to your data.
  • Upon writing your query in natural language, the application will build a SQL query and exectute the query on the database and show you the retrieved data and a brief summary of the response in natural langugae as well.
  • You can also execute manually entered SQL queries and edit the generated SQL queries and execute them again.

How to use?

  • The application is completely local, it doesn't even talk with any hosted LLM or popular LLM services online. In fact no internet connection is required to use this.
  • The application uses local LLM that will run on your local computer.
  • Download ollama and pull the model llama3.1 and make sure ollama is running with the given model.
  • Download the TalkDBApp Desktop application.
  • Run and desktop application and enter your local DB credentials and start using.

Why?

  • Learning and building SQL queries to retrive your data is quite tedious. With all the progress in LLMs, doesn't seem right to keep having to learn and write SQL queries
  • Using publicly available GPTs and other LLM services is also tedious, since we have to give context which is tedious and/or ask generic way to build SQL query and edit which is also tedious.
  • Instead context should be automatically fed and the only thing we should do is ask our questions.
  • Why not public LLMs or GPTs? Because we are all very sensitive about our data, and sharing our context/schema with public LLMs/GPTs might not be in the best interest.
  • So with this application, everything is running locally and your data/schema is secure.
  • So even if you are working with sensitite enterprise data, you can use this application without any worry.

Future

  • It is hard to imagine a future where businesses/individuals will not have the need to analyze data for better decision making.
  • It is also hard to imagine a future where a number of businesses/individuals will not be concerned with sharing their data with other parties.
  • The ultimate plan is to remove technical hurdles of communicating with your data in a secure way. Any form of insight and analysis can be done on your data using natural language by anyone without the need of any technical expertise.
